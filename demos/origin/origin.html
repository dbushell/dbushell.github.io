Typography

General typographic styles are either scoped with .prose or applied with individual classes like .h1 or .p--small. This practice stops style leaking into unexpected areas. See my article Scoping Typography CSS.

Switches

If CSS :checked selector is not supported these are normal inline checkboxes.


<label class="label--checkbox">
    <input class="switch" type="checkbox" checked> <span>Option label</span>
</label>

Inline Form

<form class="form form--inline" id="form-login">
    <ul class="form__list">
        <li class="form__item">
            <label for="form-login__username">Username</label>
            <input class="field" id="form-login__username" placeholder="Username…">
        <li class="form__item">
            <label for="form-login__password">Password</label>
            <input type="password" class="field" id="form-login__password" placeholder="Password…">
        <li class="form__item">
            <input type="submit" class="button" value="Login">
    </ul>
</form>

Single Form (full-width)

<form class="form form--single" id="form-search">
    <ul class="form__list">
        <li class="form__item">
            <label for="form-search__query">Query</label>
            <input type="text" class="field" id="form-search__query" placeholder="Enter search query…">
        </li>
        <li class="form__item">
            <button type="submit" class="button">Search</button>
        </li>
    </ul>
</form>

Responsive Grids

Grid elements should be independent of other markup. This avoids style confict and makes responsive breakpoints easier to manage. It also allows a JavaScript polyfill for browsers that don’t support box-sizing. See my article On Responsive Layout and Grids.

    <div class="gc"><p>Grid unit 2 of 2</p></div>
</div>

Columns are sized with percentage widths. This is content and design specific and not pre-defined with classes.
